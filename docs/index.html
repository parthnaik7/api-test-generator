How It Works

Six categories of tests, generated deterministically from your spec.

✓

Positive

Happy-path tests with all valid inputs and expected success responses.

✗

Negative

Missing fields, null bodies, wrong content-types, wrong HTTP methods.

◈

Boundary

Min/max values, edge string lengths, empty arrays, overflow integers, special chars.

⚿

Security

Missing tokens, expired JWTs, malformed Bearer/Basic/API-Key, SQL injection in headers.

⟨⟩

Data Type

String sent as int, boolean as string, array instead of object - all type mismatches.

⊞

Combinatorial

Pairwise combinations of optional parameters to expose interaction bugs.
